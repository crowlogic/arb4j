# AGENTS.md — arb4j

## What this project is

Java bindings to FLINT/arb (ball arithmetic with rigorous error bounds) plus a Unicode expression compiler that emits JVM bytecode computing in that ball arithmetic. Every `Real`/`Complex` carries midpoint + radius; operations propagate the radius. The expression compiler parses Unicode math notation, builds an AST, and emits typed bytecode via ASM — the output is a real JVM class, not an interpreter.

## MANDATE: Always resolve open issues

At the start of every session, check `gh issue list --repo crowlogic/arb4j --state open`. Begin working through open issues immediately. Only stop when no open issues remain or when blocked waiting on user input. Do not ask what to work on — pick the next issue and start.

## CRITICAL: Never suggest raw Java implementations

arb4j exists because Java is inferior. The expression compiler parses Unicode math notation and emits typed JVM bytecode. **NEVER suggest writing raw Java when the expression compiler can do the job.** Use `RealFunction.express("formula", ctx)` and the existing compiler infrastructure. If a program already exists (e.g. `LewisReferencePriceGenerator`), **refactor it** — do not write a new one. Refactoring means modifying the existing code, not copying it into a new file.

## CRITICAL: SWIG-generated files

Many `.java` files in `src/main/java/arb/` are **generated by SWIG** from `native/*.i` files. The header says:
> "Do not make changes to this file unless you know what you are doing - modify the SWIG interface file instead."

**NEVER edit these files directly.** Your changes will be silently overwritten on the next `make` run. They include:
- `RealPolynomial.java`, `ComplexPolynomial.java`, `Real.java`, `Complex.java`
- `arblib.java`, `arblibJNI.java`, `SWIGTYPE_p_void.java`
- All `FastDFT*.java`, `Dirichlet*.java` files
- `AlgebraicNumber.java`, `Fraction.java`, `IntegerPolynomial.java`, `RationalFunction.java`
- `RealMatrix.java`, `ComplexMatrix.java`, `Magnitude.java`, `FloatInterval.java`

**Always edit the `.i` source files** under `native/` instead, then regenerate with `make`.

Hand-written Java files (safe to edit directly): `GrMpolyNative.java`, `ComplexMultivariatePolynomial.java`, `RealMultivariatePolynomial.java`, `RealMultivariatePolynomialContext.java`, `ComplexMultivariatePolynomialContext.java`, `MultivariateOrder.java`, `NativeLibraryLoader.java`, and files under `src/main/java/arb/expressions/`.

## Build commands

```bash
# Full rebuild: SWIG codegen + C compilation → libarblib.so
make

# Run tests
mvn test

# Build jar only
mvn install

# Run a single test class
mvn test -Dtest="arb.expressions.nodes.SomeTest"

# Debug expression compiler
mvn -Ptrace test
```

`make` does everything: SWIG codegen → C compilation → shared library. The SWIG rule at `native/arb_wrap.c` depends on all `.i` files, so editing any `.i` file triggers SWIG regeneration on next `make`.

## Running Java classes

**NEVER use `mvn exec:java` or bare `java` commands.** Always use `bin/run` to set up classpath, module path, native library path, and JVM flags correctly.

```bash
bin/run fully.qualified.ClassName [args...]
```

## Architecture

### Four layers

1. **Native layer** (`native/*.i` → `arb_wrap.c` → `libarblib.so`): SWIG wraps FLINT/arb C structs into Java handles. FLINT, MPFR, GMP — all statically linked.
2. **Type layer** (`Real`, `Complex`, polynomials, matrices): Ball types carry radius. Exact types (`Integer`, `Fraction`) don't. The exact/ball distinction drives the compiler.
3. **Compiler layer** (`arb.expressions.*`): Parser → AST → ASM bytecode → `Function` instance. The codomain type determines which native operations are emitted.
4. **Function/math layer**: Typed function interfaces (`RealFunction`, `ComplexFunction`, `Sequence`), orthogonal polynomials, zeta/xi, fractional-Riccati Muntz-Pade machinery.

### SWIG call chain

```
arblib.java (static facade)  →  arblibJNI.java (native methods)  →  arb_wrap.c (C glue)  →  FLINT/arb C functions
```

Every SWIG type has `long swigCPtr` (pointer to C struct) + `boolean swigCMemOwn` (owns memory?). `getCPtr(obj)` extracts the pointer (0 for null).

### Native library loading

`NativeLibraryLoader.load("arblib")` — tries `System.loadLibrary`, falls back to extracting from classpath resource. Triggered by static initializers in every SWIG type.

### Expression compiler

- **Parser**: Hand-written recursive descent in `Expression.java`, character-by-character. Unicode operators: `Σ` `Π` `∂` `Đ` `∫` `➔` superscripts.
- **AST**: `Node<D,R,F>` hierarchy — `VariableNode`, `LiteralConstantNode`, `BinaryOperationNode` subclasses, `FunctionNode`, `IntegralNode`, `DerivativeNode`, `JetNode`, etc.
- **Codegen**: ASM library (`org.objectweb.asm`), `COMPUTE_FRAMES` mode, Java target V25. Each node has `generate(MethodVisitor, Class<?>)` that emits its own bytecode.
- **Type resolution**: `BinaryOperationNode.typeMap` maps `(leftType, rightType) → resultType` (~70 rules). `Compiler.canBeAssignedTo()` checks compatibility.
- **Optimization**: Constant hoisting (`StaticNode`), CSE (`SharedNode`), algebraic simplification.
- **Generated class structure**: Fields for intermediates (not locals), `evaluate()` with re-entrancy guard, `close()` disposes all owned fields, `initialize()` wires context variables.
- **Precision is a runtime parameter** — same compiled class works at 64 bits or 4000 bits.
- **Debugging**: `mvn -Ptrace test` enables `arb4j.trace`, `arb4j.traceNodes`, `arb4j.saveClasses`, `arb4j.decompileClasses`.

### Multivariate polynomial support

Uses FLINT's `gr_mpoly` via Java FFM (Foreign Function & Memory API) in `GrMpolyNative.java`, **not** SWIG. The `RealMultivariatePolynomial` and `ComplexMultivariatePolynomial` classes wrap `gr_mpoly_t` through this FFM bridge.

### SWIG `.i` file structure

- `native/arb.i` — main entrypoint, `%include`s everything
- `native/init.i` — FLINT includes, JNI_OnLoad, helper functions
- `native/TypesAndStructures.i` — manual struct redefinitions (can drift from FLINT headers)
- `native/Functions.i` — manually declared function prototypes

## Design patterns

### "Last argument is the result"
Every native operation writes into a caller-supplied destination and returns it: `x.add(y, bits, out)`. The two-argument form `x.add(y, bits)` writes into `this`. The bytecode emitter targets this shape.

### Deterministic native ownership; no GC safety net
SWIG finalizers are deliberately deleted. All ball types are `AutoCloseable` and must be closed. `try-with-resources` is mandatory. The generated `close()` disposes every intermediate field. `Context.closeOwnedInstances()` frees the whole cluster in one flat pass (O(1) stack depth).

### Intermediates are fields, not locals
Scratch space (loop indices, accumulators, operand buffers) are fields on the generated class, so `close()` can find and release them.

### The vector IS the scalar
A `Real` holds a contiguous native buffer of `dim` ball elements with `dim == 1` for a scalar. Java arrays of `Real` are forbidden. The type is already the vector.

### Type-directed generation, not runtime polymorphism
The compiler decides at generation time what concrete type a node produces and emits type-specific calls. No `instanceof`, no type switch survives into the generated code.

### Fail loud, never degrade
Unimplemented paths throw; they do not return null or silently fall back. For verified arithmetic, a fallback is an undocumented algorithm change.

### The `become()` pattern
`Becomable<B>.become(B that)` absorbs another object's native pointers, then sets `that.swigCMemOwn = false` so `that.close()` is a no-op. Used for returning results without allocation.

## Testing

- **JUnit 4** only (no JUnit 5). Most tests extend `TestCase` (JUnit 3 naming); a few use `@Test` annotations.
- **try-with-resources everywhere** — every `Real`/`Complex`/`Fraction` in tests is wrapped in try-with-resources.
- **LeakTracker** (`arb4j.trackLeaks=true`): Weak-reference-based native allocation leak detector. Writes to `/tmp/arb4j-leaks.txt`. Dedicated leak test isolated via Maven profile (`-Pleak-test`) because it requires `System.gc()`.
- **Surefire flags**: `-ea --illegal-native-access=allow --sun-misc-unsafe-memory-access=allow -Djava.library.path=${project.basedir} -XX:+DisableExplicitGC`
- **Test output must use SLF4J `log.debug`**, never `System.out`/`System.err` or `log.info`.

## Reference prices

Numerical pricers (e.g. `LewisReferencePriceGenerator`) are **manual CLI tools only**. Run them by hand to generate slow reference prices. Copy the output as hardcoded constants into JUnit tests. **Never call a numerical pricer from a unit test.** You cannot test the pricer with the pricer.

## Testing rules

- **NEVER filter `mvn` output** with `tail`, `head`, `grep`, or similar — run `mvn test` unfiltered and let the full output stream to the terminal. Only `tee` to file is permitted if you need a copy.
- **NEVER use `-q` (quiet) on `mvn` commands** — the build headers (project scan, phase progression, test counts) must be visible so the agent can confirm the command ran and diagnose failures from the summary lines.
- **NEVER append bare `2>&1`** — it is a useless redirect that hides stderr. Only `2>&1 | tee` is allowed (captures both streams to a file).
- **ALWAYS use `-Darb4j.verifyBytecode=true`** when running tests — this enables `CheckClassAdapter` at bytecode generation time, catching VerifyError-class defects immediately rather than at JVM load time.
- `-Dtest` permitted ONLY with `-Ptrace` for targeted diagnosis. Never change test scope/requirements.
- If a change introduces a test failure, **IMMEDIATELY revert it** — never dismiss as "pre-existing."
- No personal pronouns in analysis; third-person impersonal only.

## Known gotchas

- **Struct drift**: `native/TypesAndStructures.i` manually copies struct layouts from FLINT headers. If FLINT changes struct fields, this file must be updated manually.
- **JVM exit code 134**: SIGABRT from native code (usually SIGSEGV in libarblib.so). Check `hs_err_pid*.log` for the native stack trace.
- **Expression compiler emits `close()` on generated classes**: If you modify how intermediates are allocated, ensure the generated `close()` disposes them. The compiler tracks owned fields via `ExpressionClassLoader`.
- **Compilation ordering**: For mutually recursive functions, `compile` all but the last, then `express` the last. `express` = `compile` + instantiate + wire references.
